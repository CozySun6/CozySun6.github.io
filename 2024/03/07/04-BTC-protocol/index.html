BTC协议

Double spending attack(“花两次攻击”)

  1. 中心化方案：由央行记录每一个数字货币的归属，当有交易出现时，从央行改变货币归属。（太麻烦，中心化）
  2. 区块链方案：

两个哈希指针：

1.链接各个区块之间的，构成链表

2.指向前方某交易，是为了说明交易来源，避免DoubleSpendingAttack

地址：是由公钥推算出的

区块结构

  • Block Header(Light Node只保存header)
    • version(BTC协议版本)
    • hash of previous Block Header（上一个区块的哈希值）
    • Merkle root hash(根哈希值)
    • target（挖矿难度目标阈值）
    • nonce（随机数）
  • Block Body（Full Node包含）
    • transaction list(交易列表)

​

Distributed Consensus 分布式共识

CAP Theorem

  • Consistency
  • Availability
  • Partition tolerance

Consensus in Bitcoin 比特币内的共识协议

  • 以算力为标准而非IP地址：按nounce计算target，按照hash rate决定投票权重，
  • ForkingAttack：新节点不在最长合法链上：

​

  • 当同时出现两个节点同时接入， 该状态会维持一段时间，直到某链更长，会更新为longest valid chain并达成共识，旧链就会变为orphan block被抛弃，其出块奖励失效，无法共识。

  • block reward 出块奖励：
    • coinbase transaction：唯一比特币发行途径，记账成功者可获得该奖励，最开始为50BTC，21万个区块后，每发行一个区块只能发现25BTC，依次折半
